## 核心结论

Transformer 的目标，是逐步调整每个 token 的嵌入向量，使它不只表示单词本身，还能吸收上下文中的相关信息。

注意力机制做的事可以概括为：

1. 每个 token 先生成 `Query`、`Key`、`Value` 三类向量。
2. `Query` 与所有 `Key` 做点积，判断“我应该关注谁”。
3. 点积分数经过缩放和 `softmax`，变成注意力权重。
4. 用这些权重对 `Value` 向量加权求和，得到要添加到当前嵌入上的更新量。
5. 多个注意力头并行执行，让模型同时学习不同类型的上下文关系。

## 关键公式

标准缩放点积注意力可以写成：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V
```

其中：

- `Q`：查询向量，表示当前位置想寻找什么信息。
- `K`：键向量，表示每个位置能提供什么匹配信号。
- `V`：值向量，表示每个位置真正要传递出去的信息。
- `sqrt(d_k)`：缩放项，用于数值稳定。
- `softmax`：把任意实数分数转换为一组和为 1 的权重。

在 GPT 这类自回归模型中，还会使用掩码，让当前位置不能看到未来 token。

## 整理版笔记

### 1. 从嵌入向量到上下文含义

模型接收一段文本，并预测下一个 token。文本首先被切分成 token，然后每个 token 会通过查表得到一个高维嵌入向量。

初始嵌入只编码 token 自身，不理解上下文。例如 `mole` 可以表示“鼹鼠”“摩尔”或“痣”，但在最开始，`mole` 的嵌入向量是同一个。模型必须通过后续计算，根据上下文把它移动到更具体的语义方向上。

类似地，`tower` 本身只是一个泛化的“高大结构”概念。如果前面出现 `Eiffel`，它就应当更接近“埃菲尔铁塔、巴黎、法国、钢铁结构”等方向；如果又被 `miniature` 修饰，它还要从“大型高塔”调整为“微缩模型”。

注意力机制的作用，就是让某个 token 的嵌入从其他 token 那里吸收信息，逐步形成更精确的上下文含义。

### 2. Query：当前位置提出问题

假设句子是：

```text
a fluffy blue creature roamed the verdant forest
```

如果我们关注 `creature`，它可能需要吸收前面 `fluffy` 和 `blue` 的信息，形成“毛茸茸的蓝色生物”这一更具体的表示。

在注意力头中，每个嵌入向量会乘以一个查询矩阵 `W_Q`，生成查询向量 `q`。可以把查询理解为当前位置提出的问题，例如：

```text
我前面有没有形容词在修饰我？
```

矩阵中的权重是模型训练出来的。真实模型中的 Query 很难被人类直接解释，但用“名词寻找前置形容词”这个例子，可以直观理解它可能学到的行为。

### 3. Key：其他位置提供匹配信号

每个嵌入向量还会乘以键矩阵 `W_K`，生成键向量 `k`。Key 可以理解为每个 token 对外发出的匹配信号，例如：

```text
我是一个形容词，并且处在这个位置。
```

如果 `fluffy` 和 `blue` 的 Key 与 `creature` 的 Query 紧密对齐，它们的点积就会较大；如果是 `and`、`the` 这类无关词，点积就会较小，甚至为负。

于是，模型会得到一张“每个词关注每个词”的分数表。分数越大，表示两个位置之间的相关性越强。

### 4. 点积、softmax 与注意力模式

Query 和 Key 的点积会产生任意实数，范围可能从负无穷到正无穷。但后续要做的是加权求和，因此我们希望每一列权重都在 `0` 到 `1` 之间，并且总和为 `1`。

这一步通过 `softmax` 完成。

应用 `softmax` 后，每一列可以看成一个概率分布，表示某个目标 token 应该从其他 token 那里接收多少信息。这个归一化后的权重表，就是注意力模式。

在原始 Transformer 论文中，这部分通常压缩为：

```text
softmax(QK^T / sqrt(d_k))
```

其中除以 `sqrt(d_k)` 是为了避免点积数值随着维度变大而过大，从而让训练更加稳定。

### 5. 掩码：不能偷看未来

训练 GPT 这类模型时，模型会同时尝试预测序列中多个位置的下一个 token。这样能让一个训练样本贡献多个学习信号，训练效率更高。

但这也带来一个问题：早期 token 不能看到后续 token，否则模型就相当于提前知道答案。

因此，自回归模型会使用掩码。在应用 `softmax` 前，把所有“未来位置”的分数设为负无穷。这样经过 `softmax` 后，这些位置的权重会变成 `0`，同时每一列仍然保持归一化。

这就是 masked self-attention。

### 6. Value：真正传递的信息

Query 和 Key 决定“关注谁”，但真正被传递的信息来自 Value。

每个 token 的嵌入还会乘以值矩阵 `W_V`，得到值向量。可以把 Value 理解成：

```text
如果别人关注我，我应该向它们传递什么更新信息？
```

在前面的例子中，`fluffy` 和 `blue` 的 Value 会被较大权重加入到 `creature` 的更新中。这样，`creature` 的嵌入就会从普通的“生物”变成更具体的“毛茸茸的蓝色生物”。

更形式化地说，注意力头会生成一组更新量 `ΔE`，再把它加到原始嵌入上，得到更精细、更包含上下文的嵌入序列。

### 7. 单头注意力的完整流程

单个自注意力头可以分成四步：

1. 用 `W_Q` 为每个 token 生成 Query。
2. 用 `W_K` 为每个 token 生成 Key。
3. 通过 `QK^T`、缩放和 `softmax` 得到注意力权重。
4. 用这些权重加权 Value，并把结果加入对应位置的嵌入。

这个过程叫“自注意力”，因为 Query、Key、Value 都来自同一段输入序列。

如果 Query 和 Key 来自不同数据源，例如一种语言的文本关注另一种语言的文本，那就是交叉注意力。翻译、语音识别和多模态模型中经常会出现这种结构。

### 8. 多头注意力：并行学习多种关系

现实中的 Transformer 不只使用一个注意力头，而是并行运行很多个注意力头。每个头都有独立的 Query、Key、Value 映射，因此可以学习不同类型的关系。

一个头可能学习形容词修饰名词，另一个头可能学习代词指代，还有的头可能捕捉语义关联、语法结构、长距离依赖，甚至更抽象的语气、情感和主题。

以 GPT-3 为例：

- 嵌入维度约为 `12288`。
- 每个注意力头的 Key/Query 维度约为 `128`。
- 每层大约有 `96` 个注意力头。
- 整个模型有 `96` 层。

每个多头注意力块包含大量参数，所有层的注意力参数加起来非常庞大。但在 GPT-3 这样的模型中，注意力机制并不是全部参数来源；相当多的参数来自注意力层之间的前馈网络。

### 9. 为什么注意力机制重要

注意力机制之所以成功，不只是因为它能表达“哪些词和哪些词相关”，还因为它非常适合并行计算。

相比逐词处理的结构，Transformer 可以让一整段上下文中的 token 同时进行矩阵运算。这让它能充分利用 GPU，在短时间内处理大量数据。

过去十几年深度学习的重要经验之一是：在数据、算力和架构允许的情况下，模型规模扩大通常会显著提升能力。Transformer 的并行化特性正好让这种规模化成为可能。

## 按时间轴整理的字幕

### 0:00 - 背景：Transformer 与注意力机制

上一章开始探索 Transformer 的内部运作机制。Transformer 是大型语言模型中的关键技术之一，也是现代 AI 工具的重要组成部分。它首次因 2017 年论文《Attention Is All You Need》而广为人知。

本章重点解释注意力机制：数据在 Transformer 中如何被可视化地处理，以及为什么注意力可以让模型理解上下文。

### 0:29 - token 与嵌入向量

模型的目标是接收一段文本，并预测下一个出现的单词。输入文本会被分解成 token，通常是单词或单词片段。为了便于说明，可以先假设每个 token 都是完整单词。

Transformer 的第一步，是为每个 token 关联一个高维向量，也就是嵌入向量。这个向量可以看作单词在语义空间中的位置。

Transformer 的目标，是不断调整这些嵌入，使它们不仅编码单个单词，还融入更丰富的上下文语义。

### 1:53 - 注意力要解决什么问题

考虑 `mole` 这个词。它可以表示动物“鼹鼠”，也可以表示化学单位“摩尔”，还可以表示皮肤上的“痣”。

但在初始 token 嵌入阶段，`mole` 的向量只是查表得到的，不包含上下文。也就是说，在三种语境中，初始的 `mole` 向量是一样的。

注意力块的任务，就是根据周围词语，把这个通用向量移动到更具体的语义方向上。

### 2:46 - 例子：tower、Eiffel 与 miniature

`tower` 的初始嵌入可能只是一个泛化的高大物体。如果它紧跟在 `Eiffel` 后面，模型应当把它调整到“埃菲尔铁塔”相关方向，例如巴黎、法国、钢铁结构。

如果它又被 `miniature` 修饰，那么向量还需要进一步调整，不再强调“大型高塔”，而是更接近“微缩模型”。

更一般地说，注意力块不仅能细化单词含义，还能把一个嵌入中的信息转移到另一个嵌入中，即使两个 token 在序列中相隔很远。

### 3:39 - 序列末尾向量承载上下文

在语言模型中，预测下一个词最终依赖序列末尾的向量。假设输入是一整本悬疑小说的大部分内容，模型要预测结尾附近的下一个词，那么末尾向量就必须以某种方式编码整个上下文窗口中与预测相关的信息。

这正是注意力层和其他 Transformer 层反复更新嵌入的意义：让局部 token 逐渐吸收全局上下文。

### 4:32 - 示例句：形容词修饰名词

考虑句子：

```text
a fluffy blue creature roamed the verdant forest
```

这里先关注一种简单关系：形容词如何调整对应名词的含义。`fluffy` 和 `blue` 应该影响 `creature`，使它的嵌入表示“毛茸茸的蓝色生物”。

接下来描述的是单头注意力。真实的注意力块由多个并行头组成，但单头是理解整体机制的基础。

### 4:59 - 初始嵌入与位置编码

每个单词的初始嵌入主要编码词本身的意义。严格来说，它还会包含位置信息，也就是 token 在上下文中的位置。

位置编码的细节可以单独讨论。此处只需要知道，每个输入位置都有一个向量，模型会通过一系列计算生成新的、更上下文化的向量。

### 5:27 - 用矩阵乘法学习更新规则

注意力层中的主要计算都可以看成矩阵与向量相乘。矩阵中的元素是可训练参数，模型通过大量数据学习这些参数。

“形容词更新名词”只是为了说明注意力头可能学到的一类行为。真实模型中的注意力行为更复杂，也更难直接解释。

### 5:56 - Query：名词提出问题

可以想象，每个名词会提出一个问题：

```text
我前面有没有形容词？
```

这个问题会被编码成查询向量。计算方式是用查询矩阵 `W_Q` 乘以该词的嵌入向量。

虽然查询向量的维度通常小于嵌入向量，但它能在一个较小的空间里表达当前位置正在寻找的匹配模式。

### 6:54 - Query 矩阵作用于所有 token

同一个查询矩阵会作用于上下文中的所有嵌入，为每个 token 生成查询向量。矩阵元素都是模型参数，因此真实行为由训练数据决定。

为了便于理解，可以假设某个查询矩阵把名词映射到“寻找前序形容词”的方向。

### 7:23 - Key：回答查询

与 Query 对应的是 Key。每个嵌入向量也会乘以键矩阵 `W_K`，生成键向量。

概念上，Key 可以理解为每个 token 对查询的回答。例如形容词可能发出这样的信号：

```text
我是形容词，并且位于这个位置。
```

如果某个 Key 与某个 Query 对齐，它们的点积就会比较大，表示两个位置相关。

### 8:18 - 点积形成相关性网格

为了衡量每个 Key 与每个 Query 的匹配程度，模型会计算所有 Key-Query 对的点积。

可以把结果画成一个网格：点越大，表示相关性越强。在例子中，`fluffy` 和 `blue` 对 `creature` 的相关性应当较高；而 `and` 或其他无关词与 `creature` 的相关性应当较低。

这样就得到了一张注意力分数表，表示每个词对其他词语义更新的重要程度。

### 9:17 - softmax 得到注意力权重

原始点积分数可以是任意实数。但后续要用它们做加权求和，所以需要把它们变成权重。

对每一列应用 `softmax` 后，所有值都会落在 `0` 到 `1` 之间，并且每列总和为 `1`。这时，每一列都可以看作某个 token 对其他 token 的注意力分布。

这张归一化后的网格，就是注意力模式。

### 10:14 - 论文中的简洁写法

原始 Transformer 论文用非常简洁的形式描述了这部分：

```text
softmax(QK^T / sqrt(d_k))
```

其中 `Q` 和 `K` 是所有查询向量与键向量组成的矩阵。分母中的 `sqrt(d_k)` 用来稳定数值，避免维度变大时点积过大。

### 11:12 - 训练时为什么需要掩码

训练时，模型可以同时预测一段文本中许多位置的下一个 token。这让一个样本提供多个训练信号，效率更高。

但这要求早期 token 不能看到后续 token。否则模型会通过未来内容提前知道答案。

因此，需要把未来位置对当前 token 的影响强制设为零。

### 12:09 - 掩码的实现方式

最简单的想法是直接把未来位置的注意力权重设为零。但这样会破坏 `softmax` 后每列和为 `1` 的性质。

常见做法是在应用 `softmax` 前，把未来位置的分数设为负无穷。经过 `softmax` 后，这些位置自然变成 `0`，而剩余位置仍然保持归一化。

在 GPT 这类自回归模型中，无论训练还是生成，都需要这种掩码，防止后续 token 影响早期 token。

### 12:37 - 注意力的计算规模

注意力模式的大小与上下文长度的平方成正比。上下文越长，注意力矩阵越大，计算和内存压力也越大。

这就是为什么扩大上下文窗口一直是大型语言模型中的重要瓶颈。近年来许多注意力变体，都是为了改善长上下文下的效率。

### 13:05 - 从注意力权重到嵌入更新

注意力权重只告诉模型“谁和谁相关”。接下来还需要真正更新嵌入向量。

例如，希望 `fluffy` 的嵌入向量能对 `creature` 产生某种改变，让后者在嵌入空间中更具体地表示 `fluffy creature`。

### 13:35 - Value 矩阵

最直接的方法是引入第三个矩阵：值矩阵 `W_V`。

用 `W_V` 乘以每个词的嵌入，就得到对应的 Value 向量。Value 表示这个词如果被其他词关注，应该向对方传递什么信息。

在例子中，`fluffy` 和 `blue` 的 Value 会被大量加入到 `creature` 的更新中，而无关词的 Value 几乎不会产生影响。

### 14:36 - 加权求和得到更新量

每个 Value 向量会乘以对应的注意力权重，然后按列求和，得到每个位置的更新量。

这个更新量通常记作 `ΔE`，会被加到原始嵌入上。结果是一个更精细、更包含上下文意义的嵌入序列。

### 15:36 - 单头注意力总结

单头注意力由三个主要映射参数化：

- 查询矩阵 `W_Q`
- 键矩阵 `W_K`
- 值矩阵 `W_V`

它们共同决定每个 token 关注哪些 token，以及从这些 token 中接收什么信息。

### 16:03 - 参数规模：以 GPT-3 为例

以 GPT-3 的规模为例，嵌入维度约为 `12288`，Key 和 Query 的较小空间维度约为 `128`。

单个 Query 或 Key 矩阵大约有 `12288 × 128` 个参数，也就是约 150 万。Value 映射如果直接从大嵌入空间映射回大嵌入空间，会非常昂贵，因此实际实现中会采用更高效的分解方式。

### 17:01 - Value 映射的分解

可以把 Value 映射理解为两步：

1. 先把大嵌入向量映射到较小空间。
2. 再从较小空间映射回嵌入空间，生成真正用于更新的向量。

这种方式相当于把整体线性映射限制为低秩变换，从而减少参数量。

### 18:27 - 自注意力与交叉注意力

前面描述的是自注意力：Query、Key、Value 都来自同一段序列。

交叉注意力则用于两类不同数据之间的关联。例如翻译模型中，Key 可能来自源语言文本，Query 来自目标语言文本，注意力模式就描述两种语言中词语之间的对应关系。

在交叉注意力中通常不需要自回归掩码，因为不存在“未来 token 泄露给过去 token”的同一序列问题。

### 19:22 - 多个注意力头学习不同模式

单个注意力头只能捕捉一种或少数几种模式。真实模型会并行运行多个头，让不同头学习不同的上下文关系。

有些关系偏语法，例如形容词修饰名词；有些关系偏语义，例如某个词暗示汽车的形状与结构；还有些关系涉及指代，例如段落中的 `wizard` 可能指向 Harry Potter，而 `queen`、`Sussex`、`William` 等词可能让 Harry 更像是在指 Harry 王子。

不同关系需要不同的 Query、Key 和 Value 参数。

### 20:16 - 多头注意力块

完整的 Transformer 注意力块包含多头注意力。每个头都有独立的 Key、Query 和 Value 映射。

以 GPT-3 为例，每个块中有 `96` 个注意力头。这意味着同一层会生成 96 种不同的注意力模式，并为每个位置产生 96 组更新建议。

这些建议会被合并，并加入当前位置的嵌入，形成多头注意力块的输出。

### 21:42 - 多头注意力的直觉

多头注意力的整体思路是：让模型并行学习多种“上下文如何改变词义”的方式。

每个头提出一种可能的更新建议，所有头的结果合并后，得到更丰富的嵌入表示。

### 22:10 - 实现细节：输出投影矩阵

前面把 Value 映射描述成每个头内的一对矩阵，这是概念上容易理解的方式。实际论文和实现中，通常会把所有头的输出合并后，再通过一个大的输出投影矩阵。

因此，当资料中提到某个注意力头的 Value 矩阵时，写法可能与这里的直观解释略有不同。核心思想不变：Value 决定被关注位置实际传递给目标位置的信息。

### 23:05 - 注意力之后还有前馈网络

Transformer 中的数据流不会只经过一个注意力块。注意力之后还会经过多层感知机，也就是前馈网络。注意力和前馈网络会在很多层中反复交替执行。

随着层数加深，每个嵌入都会从其他嵌入中吸收越来越多信息，从简单词义逐渐形成更高层、更抽象的表示。

### 24:04 - 参数分布

以 GPT-3 为例，所有注意力头的参数数量非常庞大，但这仍然只占模型总参数的一部分。整个模型约有 1750 亿参数，其中很大一部分来自注意力层之间的前馈模块。

后续内容会进一步讨论这些模块和训练过程。

### 24:55 - 注意力机制为什么能成功

注意力机制成功的原因，不只在于它能捕捉词与词之间的关系，更在于它高度并行化，适合 GPU 大规模计算。

过去十几年深度学习的重要经验是：只要数据和算力允许，扩大模型规模通常能显著提升性能。Transformer 的并行架构正是让这种规模化成为现实的关键之一。
